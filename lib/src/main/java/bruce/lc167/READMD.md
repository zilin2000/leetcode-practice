# lc 167

## 注意
一开始声明right的时候得是`nums.length - 1`

这段代码实现了在一个已排序的数组 `numbers` 中查找两个数，使得它们的和等于给定的目标值 `target`。方法使用了双指针技术，其中一个指针 `slow` 从数组的开始出发，另一个指针 `fast` 从数组的末尾开始。

### 时间复杂度

- **遍历**：在最坏的情况下，`slow` 和 `fast` 指针可能需要遍历整个数组一次。`slow` 指针从数组的开始向末尾移动，`fast` 指针从数组的末尾向开始移动，直到它们满足 `slow < fast` 的条件为止。
- **总体**：因此，总的时间复杂度是 O(N)，其中 N 是数组 `numbers` 的长度。

### 空间复杂度

- **额外空间**：这段代码主要使用了两个指针 `slow` 和 `fast`，以及一些临时变量（如 `num`）来存储计算的和。这些都是常数级别的空间使用。
- **输出数组**：输出的数组不计入空间复杂度分析，因为这通常是问题的一部分要求。
- **总体**：因此，空间复杂度是 O(1)。

### 总结

这段代码通过在一个已排序的数组中使用双指针技术来查找两个数，使它们的和等于给定的目标值 `target`。它的时间复杂度是 O(N)，空间复杂度是 O(1)。这种方法利用了数组已排序的特性，通过调整 `slow` 和 `fast` 指针来有效地找到答案，无需使用额外的存储空间。
